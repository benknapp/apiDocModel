Errors and Status Codes

GRESB uses HTTP response codes to indicate success or failure of an API request. Generally, codes in the 200s indicate success, codes in the 400s indicate a users side error, and codes in the 500s indicate an error with GRESB's servers.



Code Text Description
200 OK Command was a success
400 Bad Request The request was invalid, often there is a missing parameter. An accompying error message with further information may be provided
401 Unauthorized Authentication credentials were missing or invalid. See authorization for more assistance
403 Forbidden The request was refused because your account did not recieve permission to complete this action, or your access was restricted due to usage limits
404 Not Found The requested item does not exist
500, 502, 504 Serverside Error An error has occured on our servers. Please wait a few minutes and try again
